## Decisions Made
- **Rate limiting removed**: user requested removal — not needed at current stage. Do not re-add without explicit request.
- **awaitMessageComponent pattern**: `interaction.fetchReply()` + `replyMessage.awaitMessageComponent()` is the correct pattern for collecting component interactions on ephemeral replies in discord.js v14. `interaction.channel?.createMessageComponentCollector()` silently returns `undefined` when the channel is not cached (common with ephemeral messages) and must not be used.
- **Master push rule**: NEVER push to `master` unless the user explicitly asks. Staging confirmation on `develop` first is required.
